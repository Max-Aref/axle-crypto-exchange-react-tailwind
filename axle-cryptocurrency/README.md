## üîí Security Best Practices

### For Developers

1. **Never commit sensitive data**

   - Use `.env` files for environment variables
   - Add `.env` to `.gitignore`
   - Review commits before pushing

   ```bash
   # Add to .gitignore
   .env
   .env.local
   .env.*.local
   node_modules/
   build/
   dist/
   ```

2. **Validate all inputs**

   - Server-side validation is mandatory
   - Client-side validation for UX
   - Sanitize user input to prevent XSS

   ```javascript
   // Example validation
   const validateEmail = (email) => {
     const regex = /^[^\s@]+@[^\s@]+\.[^\s@]+$/;
     return regex.test(email);
   };
   ```

3. **Use HTTPS in production**

   - All production URLs must use HTTPS
   - Implement HSTS (HTTP Strict Transport Security)
   - Install SSL certificate from trusted CA

4. **Keep dependencies updated**

   - Run security audits regularly
   - Review package updates for vulnerabilities
   - Use `npm audit` and address issues

   ```bash
   # Check for vulnerabilities
   npm audit

   # Fix known vulnerabilities automatically
   npm audit fix

   # Update all packages (with caution)
   npm update
   ```

5. **Implement security headers**

   - Content Security Policy (CSP)
   - X-Frame-Options: DENY
   - X-Content-Type-Options: nosniff
   - Referrer-Policy: strict-origin-when-cross-origin

6. **JWT Token Security**

   - Use short-lived tokens (15-30 minutes)
   - Implement refresh token rotation
   - Store tokens securely (httpOnly cookies preferred)
   - Always validate token signature

   ```javascript
   // Secure token storage
   const setAuthToken = (token) => {
     // Use httpOnly cookie (server sets)
     // Or secure localStorage for JWT
     localStorage.setItem("authToken", token);

     // Set expiry
     const decoded = parseJWT(token);
     const expiresIn = decoded.exp * 1000 - Date.now();

     // Auto-refresh before expiry
     setTimeout(() => refreshToken(), expiresIn - 60000);
   };
   ```

7. **API Rate Limiting**

   - Implement rate limiting on server
   - Set reasonable request limits per user
   - Return 429 (Too Many Requests) when exceeded

8. **Enable 2FA**
   - Encourage/require two-factor authentication
   - Support TOTP (Google Authenticator, Authy)
   - SMS as backup method

### For Users

1. **Use strong passwords**

   - Minimum 12 characters
   - Mix of uppercase, lowercase, numbers, symbols
   - Avoid common words and personal info

2. **Enable 2FA immediately**

   - Use authenticator app (not SMS if possible)
   - Save backup codes securely

3. **Use hardware wallet for large holdings**

   - Cold storage for security
   - Never share private keys

4. **Be cautious of phishing**
   - Verify URLs before entering credentials
   - Never click links from emails
   - Use browser bookmarks for login

## üêõ Troubleshooting

### Installation Issues

**Problem**: Port 3000 already in use

```bash
# Use different port on Windows/Mac/Linux
PORT=3001 npm start

# Or kill process on port 3000
# Windows:
netstat -ano | findstr :3000
taskkill /PID <PID> /F

# Mac/Linux:
lsof -i :3000
kill -9 <PID>
```

**Problem**: Module not found errors

```bash
# Clear cache and reinstall
rm -rf node_modules package-lock.json
npm cache clean --force
npm install
```

**Problem**: Styles not applying

```bash
# Rebuild Tailwind CSS
npm run build

# Or clear cache
rm -rf node_modules/.cache
npm start
```

### Runtime Issues

**Problem**: Login/Register not working

- Verify Router.js has `/login` and `/register` routes
- Check AccountBtns.js uses Link components (not `<a>` or `<button>`)
- Ensure validation functions in utils/validation.js are exported

**Problem**: Forms not validating

- Check validateEmail and validatePassword functions are imported
- Verify form handlers are calling validation functions
- Check browser console for JavaScript errors

**Problem**: Images not loading

- Verify SVG files exist in `src/assets/img/`
- Check import paths are correct (no typos)
- Ensure images are imported before use

**Problem**: CSS not loading

- Clear browser cache (Ctrl+Shift+Del)
- Run `npm start` again to rebuild Tailwind
- Check tailwind.config.js has correct content paths

### Performance Issues

**Problem**: Slow page load

```bash
# Analyze bundle size
npm run build -- --analyze

# Look for large packages to replace
npm list

# Remove unused dependencies
npm prune
```

**Problem**: High memory usage

- Close other applications
- Rebuild project: `npm run build`
- Restart development server
